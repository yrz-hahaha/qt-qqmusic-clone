> **QFileDialog 是什么**?
> QFileDialog 是 Qt 框架中用于文件和目录操作的标准对话框类，属于 QtWidgets 模块。它允许用户遍历文件系统，以便选择一个或多个文件或目录，常用于打开（Open）和保存（Save）文件功能。
> 
> **QFileDialog 中的 filter（过滤器）**：
> 过滤器就是如果我们只希望用户选择某一种类型的文件，不希望他们选一个 `.txt` 或者 `.exe` 等其他类型文件，就可以使用过滤器只保留我们想要类型的文件。
>

> 
> - 描述文案就是显示在对话框右下角下拉框里的文字，告诉用户这是什么类型；
> - 括号内容就是需要保留的文件。
> 
> 我们可以同时设置多组过滤器，当用户点击下拉菜单时，可以自己选择用那一组过滤器。比如：我们想让用户既能选音乐，也能选所有文件，就需要定义多组过滤器。然后在 Qt 中，**必须使用两个英文分号 `;;`** 来分隔不同的过滤选项。
> 
> 一个简单的例子：
> ```cpp
> // 1. 在函数中实例化对象
> QFileDialog dialog(this);
> 
> // 2. 设置对话框标题
> dialog.setWindowTitle("添加本地歌曲");
> 
> // 3. 设置文件模式：ExistingFiles 代表允许选择多个已存在的文件
> dialog.setFileMode(QFileDialog::ExistingFiles);
> 
> // 4. 设置传统过滤器（关键步骤）
> // 使用 setNameFilter 方法，在构造函数外部随时指定规则
> dialog.setNameFilter("音频文件 (*.mp3 *.wav *.flac);;所有文件 (*.*)");

> // 5. 显示对话框并判断用户是否点击了“确定”

>     }

> 
> MIME 类型的过滤器是比较现代的文件过滤方式，它会更加标准化并且跨平台兼容性更好。
> 
> MIME 类型是一种标准化的字符串，用于定义互联网上传输的文件类型、格式及内容属性。它不只是看后缀名（如 `.mp3`），而是描述文件的本质属性。
> 
> - **格式：** `类型/子类型`
> - **例子：** `text/plain`（文本）、`image/jpeg`（图片）、`audio/mpeg`（MP3 音乐）。

```cpp
void Music::parseMediaMetaData()
{
    // 1. 实例化媒体播放对象
    // `player.setMedia(musicUrl);` 的直接作用是将一个具体的媒体资源（`musicUrl` 对应的媒体资源）关联到 `QMediaPlayer` 对象上。就相当于当我们掉用这个函数后 `musicUrl` 对应的媒体资源会被加载到 player 中，我们后面就能够通过这个 player 获取这个媒体资源的数据信息
    // 并且这是一个异步机制
	//    - 当调用 `setMedia` 时，函数会立即返回，Qt 会启动一个多媒体后端，这是 Qt Multimedia 模块在底层调用的系统级服务（如 Windows 上的 DirectShow/Media Foundation，或 Linux 上的 GStreamer），这个后端服务拥有自己独立的线程，由这个独立后端服务完成打开文件流、解析文件头、提取元数据等工作
	//    - 我们的主线程会接着执行后面的代码
	// 正是因为这样一个问题，所以我们后面不能直接获取 player 中的数据信息，因为这些数据信息可能还没完全加载到 player ，我们必须要让它完全加载到 player 中才能继续读取。
	// 所以我们后面又加了一个 while 循环，让主线程不断执行这个 while 循环，直到 musicUrl 数据信息全部加载到 player 为止。
    QMediaPlayer player;
    player.setMedia(musicUrl);

    // 2. 阻塞式解析与事件分发
    // 媒体信息（如歌名、时长）的解析需要时间，我们必须等待 isMetaDataAvailable() 返回 true。
    while (!player.isMetaDataAvailable())
    {
        // 上面我们已经说过后台线程正在完成一系列解析工作，因为 setMedia 是一个异步机制，所以后台线程完成解析后，它并不能直接强行修改主线程里的 `isMetaDataAvailable()` 变量（为了线程安全）。
        // 而是完成解析后，产生一个“数据已就绪”的消息，并把这个消息丢进主线程的事件队列（Event Queue） 中。
        // 而 Qt 只有一个主线程，此时正在执行 `while` 循环，如果没有 `processEvents()` 主线程会一直卡在 `while` 循环里，根本没机会去处理事件队列中的消息
        // 有了 `processEvents()`，主线程在循环的过程中“抽空”看一眼事件队列，处理就绪队列中已就绪的事件，当解析完成后，`player` 的内部状态能被及时更新，下次 `while` 判定条件时，循环就自然结束了
        QCoreApplication::processEvents();
    }

    // 3. 提取元数据信息
    if (player.isMetaDataAvailable())
    {
        // 从多媒体容器中提取标签信息并转换为 QString
        musicName  = player.metaData("Title").toString();
        singerName = player.metaData("Author").toString();
        albumName  = player.metaData("AlbumTitle").toString();
        duration   = player.duration();

        QString fileName = musicUrl.fileName();
        int index = fileName.indexOf('-');

        if(musicName.isEmpty())
        {
            if(index != -1)
            {
                // "2002年的第一场雪 - 刀郎.mp3"
                musicName = fileName.mid(0, index).trimmed();
            }
            else
            {
                // "2002年的第一场雪.mp3"
                musicName = fileName.mid(0, fileName.indexOf('.')).trimmed();
            }
        }

        // 作者为空
        if(singerName.isEmpty())
        {
            if(index != -1)
            {
                singerName = fileName.mid(index+1, fileName.indexOf('.')-index-1).trimmed();
            }
            else
            {
                singerName = "未知歌手";
            }
        }

        // 专辑名为空
        if(albumName.isEmpty())
        {
            albumName = "未知专辑";
        }
        // 调试输出，确认解析结果
        qDebug() << musicName << " " << singerName << " " << albumName << " " << duration;
    }
}
```

将来添加到播放器中的音乐比较多，可借助⼀个类对所有的音乐进行管理。添加新 C++ 类与添加设计师界面类似。

```cpp

public:
    // 将 QQMusic ⻚⾯中读取到的⾳乐⽂件，检测是⾳乐⽂件后添加到musicList中
    void addMusicByUrl(const QList<QUrl>& urls);
    
private:
    QVector<Music> musicList;
    
/////////////////////////////////////////////////////////////////  
// musiclist.cpp 中新增
void MusicList::addMusicByUrl(const QList<QUrl> &urls)
{
    // 遍历传入的 URL 列表
    for (const auto &musicUrl : urls)
    {
        // 1. 获取文件的本地路径
        QString localFile = musicUrl.toLocalFile();

        // 2. 利用 Qt 的 MIME 数据库进行深度类型校验
        // 相比简单的后缀名判断，QMimeDatabase 能够识别文件真实的内容特征
        QMimeDatabase db;
        QMimeType mime = db.mimeTypeForFile(localFile);

        // 3. 严格筛选：仅接受 MPEG（MP3）和 FLAC 无损格式
        // 这一步是确保后续播放引擎 QMediaPlayer 能够正常工作的“防火墙”
        if (mime.name() != "audio/mpeg" && mime.name() != "audio/flac")
        {
            // 如果不是指定的音频格式，跳过该文件
            continue;
        }

        // 4. 数据落地：将合法的音乐文件存入列表容器
        // 我们在 Music 类中有一个 QUrl 的单参数构造函数，所以这里会发生隐式类型转换
        musicList.push_back(musicUrl);
    }
}
```

> **QMimeDatabase 是什么**？
> **QMimeDatabase** 是Qt 框架（Qt Core 模块）中一个功能强大的**查询工具类**。它的核心职责是：通过分析文件的扩展名、甚至直接读取文件内部的特征字节（魔数 Magic Header），识别文件的 **MIME 类型**。
> 
> **QMimeType 是什么**？
> **QMimeType** 是 Qt 框架（Qt Core 模块）中的一个数据类，用于存储 QMimeDatabase 类鉴定后的结果。一旦鉴定官（QMimeDatabase）完成了分析，它就会把所有的信息（如 MIME 名称 `audio/mpeg`、图标、描述信息等）打包进一个 QMimeType 对象中返回给我们。
> 
> 可以这样理解，QMimeDatabase 就是一个查询工具用来获取文件的信息；然后 QMimeType 用于存储 QMimeDatabase 获取的相应信息。

## 8.5 音乐分类

在 QQMusic 中，有三个显示歌曲信息的页面： 

- likePage：管理和显示点击爱心图标后收藏的歌曲 
- localPage：管理和显示本地加载的歌曲 
- recentPage：管理和显示历史播放过的歌曲

这三个页面的类型都是 CommonPage，每个页面应该维护自己页面中的歌曲。因此 CommonPage 类中需要新增：
```cpp
/////////////////////////////////////////////////////////////////  
// commonpage.h 中新增
enum PageType
{
    LIKE_PAGE, // 我喜欢⻚⾯
    LOCAL_PAGE, // 本地下载⻚⾯
    HISTORY_PAGE // 最近播放⻚⾯
};

public:
	void setMusicListType(PageType pageType);
	
private:
    void addMusicToMusicPage(MusicList &musicList);
	
private:
    // 歌单列表
    QVector<QString> musicListOfPage; // 具体某个⻚⾯的⾳乐，将来只需要存储⾳乐的 id 即可
    PageType pageType; // 标记属于likePage、localPage、recentPage哪个⻚⾯
    
/////////////////////////////////////////////////////////////////  
// commonpage.cpp 中新增
void CommonPage::setMusicListType(PageType pageType)
{
    this->pageType = pageType;
}

void CommonPage::addMusicToMusicPage(MusicList &musicList)
{ 
    // 1. 视图层重置
    // 在重新加载前清空当前页面的索引列表，确保 UI 显示与逻辑状态一致
    musicListOfPage.clear();

    // 2. 遍历全局音乐资源池
    // 使用 auto& 引用可以避免在循环中产生对象拷贝，提升处理海量歌单时的效率
    for (auto &music : musicList)
    {
        // 3. 根据页面类型执行过滤算法
        switch (pageType)
        {
        case LOCAL_PAGE:
            // 本地列表：无需过滤，直接获取所有音乐 ID
            musicListOfPage.push_back(music.getMusicId());
            break;

        case LIKE_PAGE:
            // 收藏列表：仅筛选 isLike 标记为 true 的数据项
            if (music.getIsLike())
            {
                musicListOfPage.push_back(music.getMusicId());
            }
            break;

        case HISTORY_PAGE:
            // 历史列表：仅筛选 isHistory 标记为 true 的数据项
            if (music.getIsHistory())
            {
                musicListOfPage.push_back(music.getMusicId());
            }
            break;

        default:
            // 默认兜底，防止出现未定义类型的渲染异常
            break;
        }
    }
}
```

## 8.6 更新 Music 信息到 ComonPage 界面

在歌曲分类完成之后，歌曲信息就可以更新到 CommonPage 页面了。

```cpp
/////////////////////////////////////////////////////////////////  
// commonpage.h 中新增
public:
	void reFresh(MusicList& musicList);
	
/////////////////////////////////////////////////////////////////  
// commonpage.cpp 中新增
void CommonPage::reFresh(MusicList& musicList)
{
    ui->pageMusicList->clear();

    // 1. 数据过滤：从全局资源池中提取出属于当前页面的音乐 ID 列表
    addMusicToMusicPage(musicList);

    // 2. 渲染循环：遍历过滤后的 ID 列表，动态创建 UI 控件
    for (auto musicId : musicListOfPage)
    {
        // 从全局列表中查找对应的音乐实体对象
        auto it = musicList.findMusicById(musicId);
        if (it == musicList.end())
        {
            continue; // 如果 ID 失效，跳过该项，增强鲁棒性
        }

        // 3. 核心步骤：创建自定义列表项 UI (ListItemBox)
        // ListItemBox 是你设计的包含歌名、歌手、操作按钮的自定义 QWidget
        ListItemBox* listItemBox = new ListItemBox(ui->pageMusicList);
        listItemBox->setMusicName(it->getMusicName());
        listItemBox->setSinger(it->getSingerName());
        listItemBox->setAlbumName(it->getAlbumName());
        listItemBox->setLikeIcon(it->getIsLike());

        // 4. 将自定义控件嵌入 QListWidget
        // 先创建一个标准的占位项
        QListWidgetItem* listWidgetItem = new QListWidgetItem(ui->pageMusicList);

        // 设置占位项的高度，确保能完整容纳我们的自定义控件 (固定高度 45px)
        listWidgetItem->setSizeHint(QSize(ui->pageMusicList->width(), 45));

        // 将自定义的渲染控件关联到该行
        ui->pageMusicList->setItemWidget(listWidgetItem, listItemBox);

        // 接收ListItemBox发射的setLikeMusic信号
        connect(listItemBox, &ListItemBox::setIsLike, this, [=](bool isLike){
            emit updateLikeMusic(isLike, it->getMusicId());
        });
    }

    // 5. 视图更新：强制触发界面重绘，确保所有动态生成的控件立即显示
    repaint();
}
```

```cpp
{
    // 1. 核心数据同步：定位目标歌曲并更新内存对象信息
    // 在全局 musicList 中通过唯一 ID 查找对应的歌曲实体
```
