
歌曲已经添加到程序并完成解析，解析的信息也更新到界面了，所有前置工作基本完成，接下来重点处理音乐播放，歌曲播放需要用到 Qt 提供的 QMediaPlayer 类和 QMediaPlaylist 类。

> **QMediaPlayer 类是什么**？
> QMediaPlayer 是 Qt Multimedia 模块提供的用于音频和视频播放的核心类。它作为一种高层封装，提供跨平台 API 来控制多媒体的播放、暂停、停止、进度跳转和音量调节，通常搭配 QVideoWidget 或 QMediaPlaylist 用于创建音视频播放器。
> 
> **关键特性与功能：**
> - **高级媒体功能**：支持歌曲、电影和网络收音机（流媒体）的播放。
> - **解码方式**：本身不包含解码能力，而是利用操作系统底层的媒体框架（如 Windows 的 DirectShow/Media Foundation，macOS/iOS 的 AVFoundation，Linux 的 GStreamer）。比如：当你给它一个 MP3 文件时，它会立刻向操作系统发指令：“嘿，Windows（或 Mac/Linux），我这有个音乐，你用你自带的工具帮我把它转成声音，我负责把声音传给用户。”
> - **信号与槽**：提供丰富信号，如播放状态改变（`playbackStateChanged`）、位置改变（`positionChanged`）、媒体资源切换（`sourceChanged`）等。
> - **组件搭配**：通常与 `QAudioOutput`（控制音量）和 `QVideoWidget`（渲染视频）等共同使用。 
> 
> **基本使用流程（Qt 6）**：
> ```cpp
> // 1. 创建播放器
> QMediaPlayer *player = new QMediaPlayer;
> // 2. 设置音频输出 (Qt6)
> QAudioOutput *audioOutput = new QAudioOutput;
> player->setAudioOutput(audioOutput);
> // 3. 设置视频输出
> player->setVideoOutput(videoWidget);
> // 4. 设置媒体资源并播放
> player->setSource(QUrl::fromLocalFile("video.mp4"));
> player->play();
> ```

> **QMediaPlaylist 类是什么**？
> QMediaPlaylist 是 Qt 多媒体模块（Qt Multimedia）中用于管理媒体内容播放列表的类，主要在 Qt 5 中使用。它允许用户将音频或视频文件组织成有序列表，并支持循环、随机播放等模式，通常与 QMediaPlayer 配合使用来控制播放顺序。
> 
> **主要特点和功能：**
> - **媒体管理：** 可以添加（`addMedia`）、插入、删除和移动列表中的媒体内容（以 `QMediaContent` 或 `QUrl` 表示）。
> - **播放控制：** 支持切换上一首、下一首、跳转到指定索引的歌曲。
> - **循环模式 (`PlaybackMode`)：** 支持单曲循环、列表循环、随机播放等模式。
> - **与播放器绑定：** 通过 `QMediaPlayer::setPlaylist()` 将列表绑定到播放器。
> 
> **注意**：在 **Qt 6** 中，`QMediaPlaylist` 类已被废弃。Qt 6 鼓励使用 `QMediaDevices` 或自定义列表管理来处理多媒体列表。

## 9.1 歌曲播放前置工作
### 9.1.1 播放媒体和播放列表初始化

在播放之前，需要先将 QMediaPlayer 和 QMediaPlaylist 初始化好。QQMusic类中需要添加 QMediaPlayer 和 QMediaPlaylist 的对象指针，在界面初始化时将这两个类的对象创建好。

```cpp
/////////////////////////////////////////////////////////////////  
// qqmusic.cpp 中新增
// 在 QQMusic::initUI() 函数中添加
void QQMusic::initUI()
{
	...
	
	// 初始化播放器
    initPlayer();
}

void QMusic::initPlayer()
{
    // 1. 实例化核心播放引擎
    // QMediaPlayer 负责处理播放、暂停、停止、音量调节等底层播放逻辑
    player = new QMediaPlayer(this);

    // 2. 实例化播放列表管理器
    // QMediaPlaylist 负责维护歌曲队列、切换下一曲以及管理播放模式
    playList = new QMediaPlaylist(this);

    // 3. 配置播放行为
    // 设置播放模式为“循环播放”（Loop），这是音乐播放器最常用的默认设置
    playList->setPlaybackMode(QMediaPlaylist::Loop);

    // 4. 建立逻辑关联
    // 将播放列表绑定到播放器上。此后，player 将自动根据 playList 的指示进行工作
    player->setPlaylist(playList);

    // 5. 初始化状态设定
    // 设置默认音量。注意：Qt 的音量范围通常是 0 到 100
    player->setVolume(20);
}
```

### 9.1.2 播放列表设置

播放之前，先要将歌曲加入用于播放的媒体列表，由于每个 CommonPage 页面的歌曲不同，因此 CommonPage 中新增将其页面歌曲添加到播放列表的方法，以便我们后续能够根据 CommonPage 页面不同快速切换对应页面的音乐到播放列表中。
```cpp
/////////////////////////////////////////////////////////////////  
// commonpage.h 中新增
public:
	void addMusicToPlayer(MusicList &musicList, QMediaPlaylist *playList);
	
/////////////////////////////////////////////////////////////////  
// commonpage.cpp 中新增
void CommonPage::addMusicToPlayer(MusicList &musicList, QMediaPlaylist *playList)
{
    // 1. 播放列表预处理
    // 在添加新曲目之前，通常需要清空旧的播放列表，确保播放顺序与当前页面显示一致
    playList->clear();

    // 2. 遍历全局音乐资源池
    // 建议使用 const auto &music 以提升性能，避免在循环中产生对象拷贝
    for (auto &music : musicList)
    {
        // 3. 根据当前页面类型（PageType）进行逻辑过滤
        // 只有符合当前页面属性的音乐才会被加入到播放队列中
        switch (this->pageType)
        {
        case LOCAL_PAGE:
            // 本地页面：加载所有已识别的本地音乐
            playList->addMedia(music.getMusicUrl());
            break;

        case LIKE_PAGE:
            // “我喜欢”页面：仅加载标记为 Like 的歌曲
            if (music.getIsLike())
            {
                playList->addMedia(music.getMusicUrl());
            }
            break;

        case HISTORY_PAGE:
            // “最近播放”页面：仅加载有历史记录标记的歌曲
            if (music.getIsHistory())
            {
                playList->addMedia(music.getMusicUrl());
            }
            break;

        default:
            // 兜底处理
            break;
        }
    }
}
```
### 9.1.3 播放和暂停

当我们选中播放列表中的某一首音乐，点击播放和暂停按钮时，该音乐的播放状态应该在播放和暂停之间切换。

> `QMediaPlayer::State` 枚举定义了播放器的三个核心运行状态：
> 
> - **StoppedState（停止状态 - 值为 0）**：这是播放器的“初始”或“重置”状态。
> 	- **表现为**：音频数据不再流向输出设备，且当前的播放进度（Position）通常会回到起始点。
> 	- **触发场景**：程序刚启动未播放任何内容、手动调用了 `player->stop()`，或者播放列表播放完毕。
> - **PlayingState（播放状态 - 值为 1）**：
> 	- **表现为**：播放器正在读取音频流并将其输出到音频设备。
> 	- **触发场景**：调用了 `player->play()` 且资源加载成功。
> - **PausedState（暂停状态 - 值为 2）**：
> 	- **表现为**：音频输出被挂起，但当前的播放进度（Position）会保持不变。
> 	- **触发场景**：正在播放时调用了 `player->pause()`。

```cpp
/////////////////////////////////////////////////////////////////  
// qqmusic.h 中新增
public:
	void onPlayClicked(); // 播放按钮
	
/////////////////////////////////////////////////////////////////  
// qqmusic.cpp 中新增
void QQMusic::onPlayClicked()
{
    // 调试输出，用于在控制台确认按钮点击事件是否正常触发
    qDebug() << "播放按钮点击";

    // 1. 检查播放器当前的运行状态
    // QMediaPlayer::state() 返回当前的枚举值，用于判断是正在播放、暂停还是停止
    if (player->state() == QMediaPlayer::PlayingState)
    {
        // 逻辑 A：如果歌曲正在播放中，点击按钮应执行“暂停”操作
        player->pause();
    }
    else if (player->state() == QMediaPlayer::PausedState)
    {
        // 逻辑 B：如果处于暂停状态，点击按钮应“恢复播放”
        player->play();
    }
    else if (player->state() == QMediaPlayer::StoppedState)
    {
        // 逻辑 C：如果处于停止状态（例如刚打开程序或播放结束），点击按钮开始播放
        player->play();
    }
}

// 在 QQMusic::connectSignalAndSlot() 函数中添加
void QQMusic::connectSignalAndSlot()
{
	...

    // 播放控制区的信号和槽函数关联
    connect(ui->play, &QPushButton::clicked, this, &QMusic::onPlayClicked);
}
```

当播放状态改变的时候，我们还需要修改播放按钮上图标，图片的修改可以在 onPlayCliked 函数中设置，也可以拦截 QMediaPlayer 中的 stateChanged 信号进行处理。当播放状态改变的时候，QMediaPlayer 会触发该信号。此处拦截 stateChanged 信号进行处理。

```cpp
    {
```

### 9.1.5 播放模式设置

由于我们实现播放的方式是将先将要播放的音乐加载到播放列表中，再由 QMediaPlayer 从播放列表中加载媒体源进行播放，所以我们要为音乐设置播放模式实际上就是为播放列表设计播放模式。

`QMediaPlaylist::PlaybackMode` 枚举定义了五种经典的播放模式：

|**模式名称**|**枚举值**|**行为说明**|**适用场景**|
|---|---|---|---|
|**`CurrentItemOnce`**|0|**单次播放**：播放完当前歌曲后直接停止。|适合只想听某一首特定音频时。|
|**`CurrentItemInLoop`**|1|**单曲循环**：无限重复播放当前这一首歌曲。|适合开启“洗脑模式”或者非常喜欢某首歌。|
|**`Sequential`**|2|**顺序播放**：按列表顺序播放，最后一首播完即停止。|适合希望播完歌单就让电脑安静下来的场景。|
|**`Loop`**|3|**列表循环**：按顺序播放，播完最后一首后自动跳回第一首。|音乐软件最常用的默认模式，保证音乐不断。|
|**`Random`**|4|**随机播放**：完全打乱顺序，随机选择列表中的下一首歌。|适合想保持听歌的新鲜感，打破固定顺序。|

这里我们只实现三种最常见的播放模式：循环播放、随机播放、单曲循环
```cpp
/////////////////////////////////////////////////////////////////  
// qqmusic.h 中新增
public:
	void onPlaybackModeClicked(); // 播放模式设置
	
/////////////////////////////////////////////////////////////////  
// qqmusic.cpp 中新增
void QQMusic::onPlaybackModeClicked()
{
    // 播放模式是针对播放列表 (QMediaPlaylist) 进行设置的
    // 本逻辑支持：列表循环、随机播放、单曲循环三种核心模式的循环切换
    if (playList->playbackMode() == QMediaPlaylist::Loop)
    {
        // 当前为：列表循环 -> 切换为：随机播放
        ui->playMode->setToolTip("随机播放");
        playList->setPlaybackMode(QMediaPlaylist::Random);
    }
    else if (playList->playbackMode() == QMediaPlaylist::Random)
    {
        // 当前为：随机播放 -> 切换为：单曲循环
        ui->playMode->setToolTip("单曲循环");
        playList->setPlaybackMode(QMediaPlaylist::CurrentItemInLoop);
    }
    else if (playList->playbackMode() == QMediaPlaylist::CurrentItemInLoop)
    {
        // 当前为：单曲循环 -> 切换为：列表循环
        ui->playMode->setToolTip("列表循环");
        playList->setPlaybackMode(QMediaPlaylist::Loop);
    }
    else
    {
        // 异常处理：捕获未定义的播放模式状态
        qDebug() << "播放模式错误";
    }
}

// 在 QQMusic::initPlayer() 函数中添加
void QQMusic::initPlayer()
{
	...

	// 设置播放模式
    connect(ui->playMode, &QPushButton::clicked, this, &QMusic::onPlaybackModeClicked);
}
```

当播放模式改变时按钮的图标也应该改变，这里完成图标改变的方式也有两种，一种就是直接在 onPlaybackModeClicked() 函数中进行更新；另一种就是播放模式切换时 QMediaPlaylist 会触发 playbackModeChanged 信号，我们可以为该信号关联对应槽函数中，在槽函数中完成图片切换。

这里还是选择第二种：

## 9.2 歌曲播放

我们会发现，当进行了歌曲播放的前置工作后，点击播放、模式切换、上一曲、下一曲等按钮还是没有任何变化，这是因为音乐还没有加载到播放列表，QMediaPlayer 类中还没有任何媒体源，所以很多相关信号都没有触发，点击按钮后自然没什么反应。

上述操作仅仅是在进行歌曲播放的前置工作，包括关联信号和槽，定义自定义控件中后续需要用到的接口等操作。接下来才是真正的播放操作，我们期望的是能够通过点击”播放所有“按钮和双击某一首音乐进行音乐的播放。而”播放“按钮仅仅只是暂停和继续播放当前音乐。

### 9.2.1 播放所有按钮播放

播放所有按钮属于 CommonPage 中的按钮，其对应的槽函数添加在 CommonPage 类中，但是 CommonPage 不具有音乐播放的功能，因此当点击播放所有按钮后之后，播放所有的槽函数应该发射出信号，让 QQMusic 类完成播放。

由于 likePage、localPage、recentPage 三个 CommonPage 页面都有 playAllBtn，因此该信号需要带上 PageType 参数，需要让 QQMusic 在处理该信号时，知道播放哪个页面的歌曲。
```cpp
/////////////////////////////////////////////////////////////////   
// commonpage.h 中新增
signals:
    // 该信号由QQMusic处理
    void playAll(PageType pageType);
    
/////////////////////////////////////////////////////////////////   
// commonpage.cpp 中新增
CommonPage::CommonPage(QWidget *parent) :
    QWidget(parent),
    ui(new Ui::CommonPage)
{
	...
	
	// playAllBtn按钮的信号槽处理
    // 当播放按钮点击时，发射playAll信号，播放当前⻚⾯的所有歌曲
    // playAll信号交由QQMusic中处理
    connect(ui->playAllBtn, &QPushButton::clicked, this, [=](){
        emit playAll(pageType);
    });
}
```
在 QQMusic 中，给 playAll 信号关联槽函数。在 playAll 关联的槽函数中，根据 pageType 将当前 page 页面记录下来，默认从该页面的第 0 首歌曲开始播放。
```cpp
/////////////////////////////////////////////////////////////////  
// qqmusic.h 中新增
void onPlayAll(PageType pageType);
void playAllOfCommonPage(CommonPage* commonPage, int index);

/////////////////////////////////////////////////////////////////  
// qqmusic.cpp 中新增
void QQMusic::onPlayAll(PageType pageType)
{
    // 1. 定义一个指向通用页面的指针，用于存放匹配到的页面对象
    CommonPage* page = nullptr;

    // 2. 根据传入的页面类型枚举，进行分发定位
    switch (pageType)
    {
    case PageType::LIKE_PAGE:
        // 如果是“我喜欢”页面
        page = ui->likePage;
        break;

    case PageType::LOCAL_PAGE:
        // 如果是“本地下载”页面
        page = ui->localPage;
        break;

    case PageType::HISTORY_PAGE:
        // 核心纠错：注意此处应与你定义的枚举 HISTORY_PAGE 保持拼写一致
        page = ui->recentPage;
        break;

    default:
        // 扩展接口：预留给未来可能增加的搜索页面或歌单页面
        qDebug() << "其他扩展页面播放逻辑";
        break;
    }

    // 3. 执行播放序列
    // 逻辑：从当前匹配页面的“零号索引”（即第一首歌）开始，循环播放该页面的所有曲目
    if (page != nullptr)
    {
        playAllOfCommonPage(page, 0);
    }
}

void QQMusic::playAllOfCommonPage(CommonPage *commonPage, int index)
{
    curPage = commonPage;

    // 播放page所在⻚⾯的⾳乐
    // 将播放列表先清空，否则⽆法播放当前CommonPage⻚⾯的歌曲
    // 另外：该⻚⾯⾳乐不⼀定就在播放列表中，因此需要先将该⻚⾯⾳乐添加到播放列表
    playList->clear();

    // 将当前⻚⾯歌曲添加到播放列表
    commonPage->addMusicToPlayer(musicList, playList);

    // 设置当前播放列表的索引
    playList->setCurrentIndex(index);

    // 播放
    player->play();
}
```

### 9.2.3 最近播放同步

当播放歌曲改变时，即播放的媒体源发生了变化，QMediaPlayer 会触发 metaDataAvailableChanged 信号，QMediaPlaylist 也会触发 currentIndexChanged 信号，该信号会带 index 参数，表示现在是媒体播放列表中的 index 歌曲被播放，通过 index 可以获取到 recentPage 页面中具体播放的歌曲，将该歌曲对应 Music 对象的 isHistoty 属性修改为 true，然后更新下 rencentPage 的歌曲列表，播放过的歌曲就添加到历史播放页面中了。

现在主要的问题就是如何找到这首歌曲对应的 Music 对象，因为我们有 likePage、localPage、recentPage 三个页面，这个 index 到底是哪个 CommonPage 页面中的歌曲呢？ 

我们的办法就是在 QQMusic 类中维护一个 `CommonPage*`变量 `currentPage`，记录当前正在播放的 CommonPage 页面，初始时设置为 localPage，当播放的页面发生改变时，修改 currentPage 为当前正在播放页面，其中点击播放所有按钮以及双击 QListWidget 中项的时候都回引起 currentPage 的改变。

```cpp
/////////////////////////////////////////////////////////////////   
// qqmusic.h 中新增
void onCurrentIndexChanged(int index);

/////////////////////////////////////////////////////////////////   
// qqmusic.cpp 中新增
void QQMusic::onCurrentIndexChanged(int index)
{
	// 防御性编程：拦截 clear() 等操作带来的 -1 非法索引
    if (index < 0) return;

    // 1. 获取当前播放歌曲的 ID
    // 逻辑：所有音乐的 ID 都存储在当前页面的 musicListOfPage 容器中
    // 通过索引 index 快速定位到具体的 MusicId
    const QString& musicId = curPage->getMusicIdByIndex(index);

    // 2. 在全局音乐库中查找对应的实体对象
    // 有了 MusicId 就可以通过 findMusicByMusicId 函数在 musicList 中找到该音乐
    auto it = musicList.findMusicById(musicId);

    if (it != musicList.end())
    {
        // 3. 业务逻辑处理：更新播放状态
        // 将该音乐设置为“历史播放记录”，即 isHistory = true
        it->setIsHistory(true);
    }

    // 4. UI 同步：刷新“最近播放”页面
    // 只有调用了 reFresh，用户才能在“最近播放”列表中看到刚刚听过的歌曲
    ui->recentPage->reFresh(musicList);
}

// 在 QQMusic::initPlayer() 函数中添加
void QQMusic::initPlayer()
{
	...

	connect(playList, &QMediaPlaylist::currentIndexChanged, this, &QMusic::onCurrentIndexChanged);
}

        return "";
    }
```
